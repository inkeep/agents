## üîß How This Package Is Generated and Maintained

This MCP server is **automatically generated** from the OpenAPI specification of the Inkeep Agents Evaluation API using the [Speakeasy CLI](https://www.speakeasy.com/).

### Generation Workflow

The entire generation process follows this workflow:

```
[agents-eval-api changes] ‚Üí [openapi.json update] ‚Üí [MCP server regeneration]
```

#### 1. Make Changes to the Source API

Edit files in `agents-eval-api/src/` or `packages/agents-core/src/` as needed.

#### 2. Update the OpenAPI Specification

The `openapi.json` file in this package root is fetched from the running `agents-eval-api` server:

```bash
# Manual fetch
pnpm fetch-openapi
```

Or use the **watch mode** for automatic updates during development:

```bash
# Automatically fetch OpenAPI spec when source files change
pnpm watch
```

The watch script monitors TypeScript files in:
- `agents-eval-api/src/`
- `packages/agents-core/src/`

When changes are detected, it automatically runs `fetch-openapi` to update `openapi.json`.

#### 3. Regenerate the MCP Server

With the updated `openapi.json`, use Speakeasy to regenerate the MCP server:

```bash
# Regenerate MCP server (watches openapi.json for changes)
speakeasy run --watch
```

The Speakeasy watcher will automatically regenerate the MCP server code when `openapi.json` changes.

### Complete Development Workflow

For continuous development with automatic regeneration:

1. **Terminal 1** - Run the Evaluation API server:
   ```bash
   cd agents-eval-api
   pnpm dev
   ```

2. **Terminal 2** - Watch for source changes and update OpenAPI spec:
   ```bash
   cd packages/agents-eval-api-mcp
   pnpm watch
   ```

3. **Terminal 3** - Watch for OpenAPI changes and regenerate MCP server:
   ```bash
   cd packages/agents-eval-api-mcp
   speakeasy run --watch
   ```

Now any changes to `agents-eval-api` or `agents-core` will automatically flow through to regenerate the MCP server.

### Manual Generation Steps

If you prefer manual control:

```bash
# 1. Ensure agents-eval-api is running
cd agents-eval-api && pnpm dev

# 2. Fetch the latest OpenAPI spec
cd packages/agents-eval-api-mcp
pnpm fetch-openapi

# 3. Regenerate the MCP server
speakeasy run
```

### Important Notes

- ‚ö†Ô∏è **Do not manually edit generated files** - All code in `src/` (except scripts) is generated by Speakeasy
- ‚ö†Ô∏è **Manual changes will be overwritten** on the next generation
- ‚úÖ **The `openapi.json` file is the source of truth** for generation
- ‚úÖ **Always regenerate after updating `openapi.json`**

## Development

Run locally without a published npm package:
1. Clone this repository
2. Run `npm install`
3. Run `npm run build`
4. Run `node ./bin/mcp-server.js start`

To use this local version with Cursor, Claude or other MCP Clients, you'll need to add the following config:

```json
{
  "mcpServers": {
    "InkeepEvalAPI": {
      "command": "node",
      "args": [
        "./bin/mcp-server.js",
        "start"
      ]
    }
  }
}
```

Or to debug the MCP server locally, use the official MCP Inspector: 

```bash
npx @modelcontextprotocol/inspector node ./bin/mcp-server.js start
```
