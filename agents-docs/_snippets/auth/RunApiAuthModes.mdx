## Authentication

Choose the authentication method:

<Tabs>
<Tab title="App Credential (recommended)">

App credentials are the recommended auth method for browser-facing deployments. The App ID is safe to expose client-side.

**Setup:**
1. [Create an app credential](/talk-to-your-agents/app-credentials#create-an-app) for your project
2. Request an anonymous session token from your client
3. Use the token and App ID in chat requests

**Request Headers:**
```http
Authorization: Bearer <session_token>
X-Inkeep-App-Id: <app_id>
```

See [App Credentials](/talk-to-your-agents/app-credentials) for the full auth flow details.

</Tab>

<Tab title="API Key (server-to-server)">

Use API keys for server-to-server integrations where the secret is never exposed to end-users.

**Create an API Key:**
1. Open the Visual Builder Dashboard
2. Go to your Project → **API Keys**
3. Click **Create**, select the target agent
4. Copy the API key (it will be shown once) and store it securely

**Request Header:**
```http
Authorization: Bearer <api_key>
```

<Warning>
API keys are secrets. Never expose them in client-side code or browser requests.
</Warning>

</Tab>

<Tab title="Development (no auth)">

When running the API server locally with `pnpm dev`, authentication is automatically bypassed. You can use headers in the request instead:

**Request Headers:**
```http
x-inkeep-tenant-id: <tenant-id>
x-inkeep-project-id: <project-id>
x-inkeep-agent-id: <agent-id>
```

<Warning>
This mode is for development only. Never use in production as it bypasses all security checks.
</Warning>

</Tab>

</Tabs>

See [Authentication → Run API](/api-reference/authentication/run-api) for more details.
