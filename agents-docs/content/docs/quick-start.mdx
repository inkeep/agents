# Install the Inkeep CLI

The Inkeep CLI is the primary tool for deploying and managing your agents. Install it globally to get started:

### Step 1: Install the CLI

```bash
# Using npm
npm install -g @inkeep/agents-cli

# Using pnpm  
pnpm add -g @inkeep/agents-cli

# Using yarn
yarn global add @inkeep/agents-cli
```

### Step 2: Verify installation

```bash
inkeep --version
```

### Step 3: Initialize your project

Create a new directory for your agents and initialize the configuration:

```bash
mkdir my-agents
cd my-agents
inkeep init
```

This will create an `inkeep.config.ts` file with your project settings.

# Create Your First Agent

### Step 1: Create an agent configuration

Create a new file `my-first-agent.ts`:

```typescript
import { agent, agentGraph } from '@inkeep/agents-sdk';

const assistantAgent = agent({
  id: 'assistant',
  name: 'Assistant Agent',
  description: 'A helpful assistant that can answer questions',
  prompt: 'You are a helpful assistant. Answer questions clearly and concisely.',
});

export const graph = agentGraph({
  id: 'my-first-graph',
  name: 'My First Agent Graph',
  description: 'A simple agent graph with one assistant',
  defaultAgent: assistantAgent,
  agents: () => [assistantAgent],
});
```

### Step 2: Deploy your agent

Push your agent configuration to the server:

```bash
inkeep push my-first-agent.ts
```

### Step 3: Chat with your agent

Start an interactive chat session:

```bash
inkeep chat my-first-graph
```

### Step 4: List your agents

View all deployed agent graphs:

```bash
inkeep list-graphs
```

# Launch the Visual Builder (Self-Hosted)

If you want to run the Visual Builder locally for development or self-hosting:

### Step 1: Clone the repository

```
git clone https://github.com/inkeep/agents.git
cd agents
```

### Step 2: Configure environment variables

```bash
cp agents-manage-api/.env.example agents-manage-api/.env && cp agents-run-api/.env.example agents-run-api/.env && cp agents-manage-ui/.env.example agents-manage-ui/.env && cp examples/.env.example examples/.env
```

Add your `ANTHROPIC_API_KEY` to an `.env` file at `/agents-run-api/`
```
ANTHROPIC_API_KEY=sk-ant-xyz789
```

### Step 3: Setup and install
```bash
pnpm install
```

### Step 4: Initialize database
```bash
pnpm --dir ./packages/agents-core db:push
```

### Step 5: Run the agent framework
```bash
pnpm dev
```

### Step 6: Start building!
Open `http://localhost:3000` in the browser and start building agents.

# Push your first graph

A key advantage of the Inkeep agent framework is its seamless code-to-visual workflow: define your agents programmatically, push them to the visual builder, and continue developing with the intuitive drag-and-drop interface.

Follow the steps below to push your first graph using the [Inkeep CLI](/typescript-sdk/cli-reference).

### Step 1: Download the Inkeep CLI

```bash
npm install -g @inkeep/agents-cli
```

### Step 2: Push a graph

```bash
cd examples
inkeep push basic.graph.ts
```

<Note>The graph will be pushed to whichever project is specified in the `/examples/inkeep.config.ts` file, which is set to the `default` project by default. Feel free to change it to your own project.</Note>

# Set up live traces

### Step 1: Launch docker containers
```bash
cd deploy/docker
docker compose up -d
```

### Step 2: Fetch SigNoz API Key
Open `http://localhost:3080` in the browser. Go to **Settings** → **Workspace Settings** → **API Keys** and copy the API key.

### Step 3: Configure environment variable
Create a `.env` file at `/agents-manage-ui/` with the following variable:

```
SIGNOZ_API_KEY=your-signoz-api-key-here
```

# Set up credentials

### Step 1: Create .env file and generate encryption key 

```bash
cp deploy/docker/.env.nango.example deploy/docker/.env && encryption_key=$(openssl rand -base64 32) && sed -i '' "s|REPLACE_WITH_BASE64_256BIT_ENCRYPTION_KEY|$encryption_key|" deploy/docker/.env && echo "Docker environment file created with auto-generated encryption key"
```

### Step 2: Restart the containers

```bash
cd deploy/docker
docker compose up -d
```

### Step 3: Get your Nango API key
Open the Nango Dashboard: `http://localhost:3050` and navigate to **Environment Settings** → **API Keys** and copy the API key.

### Step 4: Configure environment variables
Paste the following command and enter your Nango API key when prompted:

```bash
printf "Enter your Nango API key: " && read key && sed -i '' "s|^NANGO_SECRET_KEY=.*|NANGO_SECRET_KEY=$key|" agents-manage-api/.env agents-run-api/.env agents-manage-ui/.env && echo "Application files updated with Nango API key"
```

### Step 5: Start creating credentials!

Navigate to the Credentials tab in the left sidebar and click "Create credential".
